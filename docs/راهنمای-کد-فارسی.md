# راهنمای خواندن کد — سامانه مدیریت فرش

این سند برای کسی است که کدنویسی بلد نیست ولی می‌خواهد بداند **چه چیزی کجا و چرا** اتفاق می‌افتد. روی هر فایل منبع هم توضیح فارسی (`///` یا `//`) گذاشته شده است.

## جریان کلی یک درخواست

```
مرورگر (Vue)  →  API (Controllers)  →  سرویس (Application)  →  دیتابیس (Infrastructure)
```

مثال: «تکمیل مرحله قالیشویی فرش»

1. کاربر در `RugDetailView.vue` دکمهٔ تکمیل را می‌زند.
2. `api.post('/rugs/.../advance')` با توکن JWT فرستاده می‌شود.
3. `RugsController` شناسه کارگاه را از توکن می‌خواند.
4. `RugManagementService` فرش را از دیتابیس بارگذاری می‌کند.
5. `WorkflowEngine.AdvanceStepAsync` وضعیت مرحله را Completed می‌کند و مرحله بعد را InProgress می‌کند.
6. `SaveChanges` در PostgreSQL ذخیره می‌کند.
7. پاسخ JSON (RugDto) به Vue برمی‌گردد و صفحه به‌روز می‌شود.

## لایه‌ها (پوشه `src/`)

| پوشه | نقش | مثال |
|------|-----|------|
| **Domain** | موجودیت‌ها و قوانین ثابت دامنه | `Rug`, `WorkflowTemplate` |
| **Application** | منطق کار؛ بدون SQL مستقیم | `WorkflowEngine`, `CostCalculationService` |
| **Infrastructure** | EF Core، JWT، Repository | `AppDbContext`, `RugRepository` |
| **Api** | HTTP و امنیت | `RugsController`, `Program.cs` |

## مفاهیم مهم

- **Tenant (کارگاه)**: هر مشتری شما یک کارگاه جدا؛ فرش و قالب‌ها فقط برای همان کارگاه.
- **ProcessStepType**: نوع کار (قالیشویی، دارکشی، …) — در کل سیستم یکسان.
- **WorkflowTemplate**: ترتیب مراحل از پیش تعریف‌شده برای کارگاه.
- **RugWorkflowStep**: نمونهٔ واقعی یک مرحله روی یک فرش مشخص.
- **WorkflowEngine**: تعیین می‌کند کدام مرحله فعال است و هزینه چطور جمع می‌شود.

## فرانت (`client/src/`)

| پوشه | نقش |
|------|-----|
| `api/client.ts` | ارتباط با سرور + توکن |
| `stores/auth.ts` | وضعیت ورود |
| `router/index.ts` | آدرس صفحات و محافظت login |
| `views/` | صفحات اصلی |
| `components/` | قطعات تکرارشونده (مثل نوار مراحل) |
